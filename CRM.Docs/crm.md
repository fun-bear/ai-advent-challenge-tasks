# CRM: описание клиентских данных

## Назначение данных

В CRM хранится карточка пользователя с базовыми атрибутами, которые нужны для обслуживания, контроля доступа, финансового учёта и аналитики.

Основные поля:

- `id` — уникальный идентификатор клиента;
- `fullName` — ФИО клиента;
- `isBlocked` — статус блокировки (`true`/`false`);
- `role` — роль пользователя в системе;
- `balance` — финансовый баланс клиента.

---

## Роли в системе

По текущему набору данных используются следующие роли:

- **User** — базовый пользователь CRM, работает со своими операциями и данными.
- **Manager** — менеджер, может сопровождать клиентов, контролировать статусы и процессы.
- **Support** — сотрудник поддержки, обрабатывает обращения и помогает по вопросам сервиса.
- **Admin** — администратор с расширенными правами управления системой.
- **Analyst** — аналитик, использует агрегированные данные для отчётности и выводов.
- **Auditor** — аудитор, проверяет корректность операций и состояние учётных данных.

---

## Блокировка пользователя

Пользователь **может быть заблокирован** (`isBlocked: true`).

Возможные причины блокировки (вымышленные правила для CRM):

1. Подозрение на мошеннические действия по аккаунту.
2. Нарушение правил использования сервиса.
3. Неуспешная верификация личности.
4. Длительная неактивность с последующей авто-блокировкой для безопасности.
5. Критическая задолженность или спорные финансовые операции.

Разблокировка обычно проводится после проверки службой поддержки или ответственным менеджером.

---

## Баланс пользователя

Баланс может быть:

- **положительным** (например, `15200`, `48300`, `120000`),
- **нулевым** (`0`),
- **отрицательным** (например, `-150`, если есть задолженность или корректировка).

По текущим данным:

- минимальное значение баланса: **-150**;
- максимальное значение баланса: **120000**.

Баланс используется для контроля расчётов, задолженности, лимитов и внутренней отчётности.

---

## Пример интерпретации карточки

Типовая карточка клиента в CRM отражает:

1. кто это (`fullName`),
2. какой у него доступ (`role`),
3. активен ли аккаунт (`isBlocked`),
4. текущее финансовое состояние (`balance`).

Такой формат помогает быстро принимать операционные решения: допускать пользователя к операциям, передавать кейс в поддержку, эскалировать на аудит или финансовый контроль.
